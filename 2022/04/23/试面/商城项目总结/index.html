项目介绍与架构

SpringBoot + SpringCloud + mysql + MyBatis-Plus + Redis + RabbitMQ + ES

B2C模式的在线销售平台。
商城分为后台管理部分与用户前台使用部分。
后台管理部分包括：
商品管理模块（商品分类、品牌、平台属性、SPU与SKU以及销售属性、商品上下架和商品评论管理等）、 库存管理模块、 订单管理模块、 促销管理（秒杀等商品设置）、 客户模块 统计报表模块和系统基础权限等模块。

用户前台使用部分：
商城首页、 商品搜索（可按条件查询展示）、 商品详情信息展示、 购物车、 用户单点登录和社交登录（微信登录）、 用户会员中心、 订单的创建修改、 展示以及在线支付（支付宝、微信）、 物流模块、 商品评论以及秒杀活动等功能

商品管理模块

商品管理是怎么做的

  1. 多级分类功能: 电器->电视,空调->4k电视,曲面电视
  2. 品牌管理: 用于搜索的的关键字
  3. 产品单元管理: 商品特性的集合 iphone就是一个产品单元
  4. 产品统一编号管理：一个128g，银色的，支持5g的iphone就是一个统一编号的产品,也是库存单元记录数

商品管理中的技术

  1. 将上架的商品缓存到redis中,下架的商品从redis中移除
  2. 将商品信息同步到es用于搜索
  3. 将后台商品的图片使用FastDfs进行分布式存储

商城首页与搜索

商城首页怎么做的

  1. 首页分类
  2. 搜索栏，
  3. 商品列表展示
  4. 根据热度值排名商品

    商品首页的技术

  5. 数据库视图
  6. 页面静态化
  7. es全文检索

库存管理

库存管理是怎么做的

是后台管理项目

  1. 查询库存
  2. 查询供应商
  3. 客户管理
  4. 库存警告
  5. 报损报溢管理

使用的技术

普通的增删查改

购物车

购物车怎么做的

  1. 非登陆状态添加商品到购物车(localStorage保存到浏览器，不浪费服务器资源)
  2. 用户登陆状态，添加商品到redis，采用的hash结构存储
  3. 登陆后合并购物车,追加到redis，清空cookie(设置存活时间)
  4. 修改购物车勾选状态
  5. 验证库存，和验证价格，做数据替换

下单

下单怎么做的

  1. 高热度值商品通过定时任务在redis中预热数据,通过Scheduled设置定时任务
  2. 用户下单排队，生成排队消息，给mq发送请求，监听者收到下单消息再生成订单

实现技术

库存超卖:通过redis队列的原子性，设置队列长度为库存数量，下单就rightPop,回滚库存就leftPush，当队列为空就表示售罄
详情页静态化:将动态渲染的页面保存为静态页面（一般存储在nginx），提高访问速度(Thymeleaf的TemplateEngine)
商品售罄：发送消息，修改商品状态
防止用户重复下单:创建订单的时候，用订单信息计算一个哈希值，判断redis中是否有key，有则不允许重复提交，没有则生成一个新key，放到redis中设置个过期时间，然后创建订单

幂等性

幂等性指的就是指一次和多次请求某一个资源应该具有同样的副作用。
